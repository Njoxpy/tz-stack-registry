# tz-stack-registry

A community-sourced registry mapping the **technical landscape of Tanzanian companies** — tech stacks, tooling choices, and interview patterns.

!!! info "What this is"
    A structured, verifiable, community-maintained map of how Tanzanian companies build software. Each entry is backed by a public source (job post, engineering blog, conference talk) or a properly anonymized testimony.

## Why It Exists

Tanzanian developers lose time guessing what to learn and how to prepare for interviews. This registry centralizes that intelligence so that:

- Developers can align learning with actual market demand.
- Candidates can prepare for interviews using real, sector-specific patterns.
- The ecosystem gains visibility into how local companies build software.

## How to Use It

1. **Pick a sector** (Fintech, Agritech, Logistics, SaaS) in the registry repo.
2. **Open a company folder** to read its `stack.json` and `interview-questions.md`.
3. **Follow the [Interview Guidelines](interview-guidelines.md)** to turn the data into a preparation plan.

## How to Contribute

- Read the [Contribution Standards](contribution-standards.md) — the "Elite Standard" for what gets merged.
- Read the [Privacy Policy](privacy-policy.md) before submitting any interview testimony.
- Read the [Code of Conduct](code-of-conduct.md) — expectations for everyone in the project.
- Open a pull request. Every claim must include a verifiable source.

## Disclaimer

All data in this registry is **community-sourced**. Stacks evolve; job posts and blogs reflect a point in time, not a guarantee of current practice. Treat every entry as a strong signal, not ground truth. Companies named in the registry are not affiliated with this project and have not endorsed its contents.
